打造你的 未來感個人品牌 Hero 區

未來這一區會放上一句話自我介紹、清楚的核心職稱（例如 Frontend Developer）、以及引導訪客瀏覽作品集或下載履歷的主要行動呼籲按鈕。現在看到的文字、按鈕與排版皆採科技感暗色風格的占位設計，方便之後整段替換。

目前仍是「Phase 1：環境與骨架」，專注在布局、RWD 與插槽規劃。正式文案、個人形象照、CTA 文案與更細緻的動畫或特效，將在之後使用 Section SDD 的 Prompt 產生，並以整個 section 的方式貼回。

預計主行動呼籲：瀏覽作品集（占位） 次行動呼籲：查看聯絡方式（占位）

以上按鈕目前為示意，未來可依品牌調性改成「下載履歷」、「發送合作邀請」或其他你想要突出的轉換行為。

之後會用 Section SDD 的 Prompt 產生新的 Hero section，請把新的 <section id="hero" ...>...</section> 程式碼整段貼在 SECTION:HERO-START 與 SECTION:HERO-END 這兩個註解中間，完全取代現在這一區。你不需要逐段修改，只需一次性覆蓋整個 section 即可。

Section · About 區科技感占位

關於我 （之後會用 Section SDD 替換）

這一區預計用來介紹你的背景、工作經驗、專業領域與個人特質，並延續整體科技感暗色主題。現在的文案與區塊只是一個「結構樣板」，協助你在 Phase 1 先建立清楚的插槽，方便未來以 Section SDD 產生正式 About 區後完整覆蓋。

目前 About 區內容為占位描述（技術視角）

這裡可以暫時說明未來預期呈現的資訊類型，例如：你作為前端工程師的年資、曾經參與的專案規模、偏好的技術堆疊，以及你如何看待使用者體驗與效能優化。實際文案會在 Section SDD 階段依照需求產出更貼近你個人風格的內容。

之後會用 Section SDD 的 Prompt 產生新的 About section，請把新的 <section id="about" ...>...</section> 程式碼整段貼在 SECTION:ABOUT-START 與 SECTION:ABOUT-END 這兩個註解中間，完全取代現在這一區。你不需要考慮對齊每一段，只需確保新的 section 保留 id="about" 即可。

給開發者的插槽說明（版面與資訊結構）

你可以把這個區塊視為科技感資訊側欄，未來可放時間軸、關鍵數字（例如「專案數量」、「上線服務人次」）、或個人價值觀與職涯願景。此處先以簡單文案作為占位，目的是提示「這個位置未來可以放更多結構化內容」。

當你取得 Section SDD 產生的新 About 區程式碼時，直接以整個 <section id="about" ...>...</section> 覆蓋 SECTION:ABOUT 註解之間的內容即可。若你希望維持現在的科技感卡片風格，只要延用 .about-block 等 class 命名，就能沿用同一組 CSS 樣式。

提示：About 區目前採單欄為主，寬螢幕時則變為左右雙欄科技感卡片。未來若 Section SDD 產出的版型完全不同，也可以自由調整，只要保留 id="about" 與必要 class，並在 CSS:ABOUT 區塊中補上或覆寫樣式即可。

提示：目前 Skills 區已預留 .skills-group、.skill-tag 等 class 名稱，方便你在之後為科技感技能標籤加入更多特效（例如漸層邊框、發光或動畫），同時維持「不使用 ID 選擇器」的規範。

給開發者的實作備註（科技感版）

若未來想將這一區改成實際的聯絡表單（例如姓名、Email、訊息內容等欄位），可以在 Section SDD 中產生對應的 <form> 結構。由於本階段僅使用 HTML 與 CSS，並未綁定任何 JavaScript，你可以自由選擇之後要使用後端、第三方服務或前端框架來處理送出行為。

如果你偏好維持「靜態科技感聯絡資訊清單」的設計，也可以只更新文字與連結本身，不必重寫整個 section。兩種方式皆與此骨架完全相容。

提示：所有聯絡連結的 href 皆已填入合法格式（mailto:、tel: 與 https URL），確保在任何階段都不會出現空值。當 Section SDD 產生新的 Contact 區時，只需確保保留 id="contact"，並將程式碼放在 SECTION:CONTACT 註解之間即可。
